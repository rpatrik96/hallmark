# HALLMARK v1.1 Expansion — Brainstorm

Captured 2026-05-07. Branch: `expand-v1.1`.

## Context

Post-deadline status: paper out, benchmark v1.0 frozen at 2,525 entries. Reviewers
will likely flag two weaknesses:
1. **Temporal staleness** — valid entries cluster in 2021–2023; no 2024–2025 coverage despite scraper supporting it.
2. **Domain-monoculture** — every entry comes from ML conferences (NeurIPS/ICML/ICLR/AAAI/CVPR + ML journals). No evidence detectors generalize beyond their training distribution.

Goal of v1.1: address both without invalidating v1.0 results. Strategy: **add new
splits, do not modify existing ones.**

## Tier A — confirmed for v1.1

### A1. Temporal extension (`*_2024_2025` add-on)
- Re-run DBLP scraper with `--years 2024 2025` over existing ML venues.
- Apply same perturbation pipeline; no new generators needed.
- Surfaces "are detectors stale by 2 years?" question — directly addresses likely review point.
- Cost: hours of API time, ~$0 inference.

### A2. Cross-domain test split (`test_crossdomain.jsonl`, ~500 entries)
Three sub-corpora, evaluation-only (no dev/train):

| Sub-corpus       | Source                   | Target |
|------------------|--------------------------|--------|
| Bio/life-sci     | bioRxiv API + CrossRef   | ~150   |
| CS-non-ML        | DBLP (SIGIR/KDD/WWW/SIGGRAPH) | ~200   |
| Medical/clinical | PubMed E-utilities       | ~150   |

Validates that detectors aren't overfit to the ML citation manifold (DBLP venue
keys, arXiv preprint norms, ML author network).

### A3. Venue-key registry refactor
`DBLP_VENUE_KEYS` is hard-coded ML-centric in `scraper.py`. Promote to a module
constant grouped by domain so cross-domain scraping is data, not code.

## Tier B — promising, needs scoping

### B1. Domain-specific hallucination types
- `wrong_pmid` / `pmid_doi_swap` for bio/medical
- `pdb_id_hallucination` for structural biology
- `mendeley_year_drift` for soc-sci (typical in author-year cites)

Wire only into the cross-domain split if signal is interesting.

### B2. Real-world incident corpus expansion
- bioRxiv exposes a "withdrawn" feed → curated retracted-paper signal
- Retraction Watch DB → high-credibility ground truth for hallucinated cites in published work
- Estimated yield: 50–150 strong real_world entries

### B3. Coverage probe via OpenAlex
Sample 2024–2025 papers across domains; measure how many citations from those
papers our scraped pool covers. Reportable as appendix table.

## Tier C — defer
