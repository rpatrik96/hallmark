# HALLMARK v1.1 Expansion ‚Äî Brainstorm

Captured 2026-05-07. Branch: `expand-v1.1`.

## Context

Post-deadline status: paper out, benchmark v1.0 frozen at 2,525 entries. Reviewers
will likely flag two weaknesses:
1. **Temporal staleness** ‚Äî valid entries cluster in 2021‚Äì2023; no 2024‚Äì2025 coverage despite scraper supporting it.
2. **Domain-monoculture** ‚Äî every entry comes from ML conferences (NeurIPS/ICML/ICLR/AAAI/CVPR + ML journals). No evidence detectors generalize beyond their training distribution.

Goal of v1.1: address both without invalidating v1.0 results. Strategy: **add new
splits, do not modify existing ones.**

## Tier A ‚Äî confirmed for v1.1

### A1. Temporal extension ‚Äî merged into existing `temporal_supplement`

The canonical 2024‚Äì2025 ML temporal corpus lives at
`results/temporal_supplement/temporal_supplement_2024_2025.jsonl` (originally
created Apr 22, 448 entries, 300 valid + 148 hallucinated). It already has
pre-computed baseline evaluations for 10+ LLMs (Sonnet 4.6, Opus 4.7,
Gemini Pro/Flash, DeepSeek R1/V3, Qwen, Mistral, Llama 4 Maverick, GPT-5.1)
under `results/temporal_supplement/llm_*_temporal_*.json`. Helper scripts:
`generate_2026_entries.py`, `evaluate_temporal_supplement.py`,
`probe_temporal_robustness.py`.

The May 2026 re-scrape produced an independent 450-entry sample with more
even hallucination type coverage (the original was sparse on
`swapped_authors` and `merged_citation`). After comparing, we **merged**
the new entries into the existing file rather than maintaining two
parallel corpora:

| | Pre-merge | Post-merge |
|---|---|---|
| Total | 448 | **858** |
| Valid | 300 | 440 |
| Hallucinated | 148 | **418** |
| `swapped_authors` | 3 | **27** |
| `merged_citation` | 1 | **22** |

Merge logic (`scripts/merge_temporal_supplement.py`):
- Existing entries kept verbatim (preserved `bibtex_key` ‚Üí existing eval
  JSONs remain valid for that subset).
- New entries' DOIs deduped against existing (40 valid skipped). Hallucinated
  entries kept regardless of DOI presence ‚Äî most have intentionally damaged
  DOIs by design (fabricated_doi etc.).
- New entries re-keyed with the corrected DBLP scheme
  (real surname + year + first-word + 6-char SHA-1) to avoid the
  "0001"-disambiguator bug that contaminated the original keys.

Baseline evals on the new 410 entries will need to be computed when budget
allows; existing evals on the original 448-entry subset remain valid.

The withdrawn-incidents file is a **reusable data source**, not a standalone
split ‚Äî analogous to the existing `real_world_incidents.jsonl` and
`gptzero_neurips2025.jsonl`. Downstream split builders may sample from it.
Each entry has a real DOI, real title, and real authors ‚Äî but the paper
has been formally withdrawn, so citing it is a real-world failure mode no
perturbation generator produces. Tagged `generation_method=real_world`,
`hallucination_type=plausible_fabrication`,
`subtests.cross_db_agreement=False` (everything else True ‚Äî that's the trap).

Both splits share the same 14-type hallucination taxonomy as v1.0 and use the
same tier1/tier2/tier3 perturbation generators, so direct evaluation
comparison against v1.0 is well-defined.

## Bugs fixed in passing

- DBLP author records can include `0001`-style homonym disambiguator
  suffixes; `dblp_hit_to_entry` now strips them via `_clean_dblp_author`.
  v1.0 was unaffected (0 entries had artifacts), but newer scrapes did.
- `dblp_hit_to_entry` was generating colliding bibtex_keys when the same
  surname published multiple papers starting with the same word in the
  same year (e.g., 6 collisions across 480 NeurIPS/ICML/ICLR 2024 entries).
  Now appends a 6-char SHA-1 digest suffix derived from title+authors+DOI.
