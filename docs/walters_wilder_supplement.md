# Walters & Wilder ChatGPT-citation supplement ‚Äî dataset & bibtexupdater analysis

This document describes the `supplement_chatgpt_citations` eval-only split and a
full `bibtexupdater` evaluation on it. Machine-generated metric tables are
regenerated by `scripts/analyze_walters_wilder.py` into `results/walters_wilder/`.

## 1. Source dataset

William H. Walters & Esther Isabelle Wilder, **"Fabrication and errors in the
bibliographic citations generated by ChatGPT"**, *Scientific Reports* 13:14045
(2023), [doi:10.1038/s41598-023-41032-5](https://www.nature.com/articles/s41598-023-41032-5)
(CC BY 4.0). The coded ground truth is Supplementary Appendix 3 (sheet "Works
cited in those papers", 636 rows).

The authors prompted ChatGPT-3.5 and GPT-4 to write 84 short literature reviews
across 42 multidisciplinary topics, then hand-verified every one of the 636
resulting citations, coding each for (a) whether the *work itself* is fabricated
and (b) substantive metadata errors on real works (wrong author, title, year,
journal, volume/pages). Reported fabrication rates: **GPT-3.5 55%**, **GPT-4 18%**.

## 2. Why this is a meaningful HALLMARK extension

HALLMARK's v1.0 splits are almost entirely ML/CS-conference papers produced by
perturbation or scraping. This dataset adds two axes the benchmark is thin on:

1. **Authentic LLM hallucinations** ‚Äî real ChatGPT output with expert labels,
   not synthetic perturbations. (v1.0 test has only ~90 `llm_generated` +
   ~34 `real_world` entries.)
2. **Multidisciplinary subject matter** ‚Äî humanities, social sciences, and
   natural sciences, versus HALLMARK's CS/ML focus. It is the natural companion
   to the v1.1 cross-domain split.

## 3. Conversion methodology

Only the **supported publication type ‚Äî journal articles** ‚Äî is kept. Books,
chapters and websites (‚âà32% of the source) are out of scope because DOI /
CrossRef / DBLP / OpenAlex-based verifiers cannot resolve them (in a probe run,
100% of books returned `skipped`).

APA reference strings are parsed into BibTeX fields with a parser hardened for
this corpus's humanities-heavy style: titles containing colons, embedded quotes
or a terminal `?`; APA ellipsis / *et al.* author truncation; and a Chicago-style
bare-year fallback. A quality guard drops any residual broken parse.

### Label mapping (faithful to the HALLMARK taxonomy)

A field mismatch on a real work **is** a hallucination and is mapped to the
matching HALLMARK type rather than left VALID:

| Source coding | HALLMARK type | Tier | n |
|---|---|---|---|
| work itself fabricated | `plausible_fabrication` | 3 | 139 |
| real work, wrong title | `near_miss_title` | 3 | 13 |
| real work, wrong authorship | `author_mismatch` (`swapped_authors`) | 2 | 12 |
| real work, wrong/invented journal | `wrong_venue` | 2 | 5 |
| real work, no substantive error | VALID | ‚Äî | 172 |

When several fields are wrong, the most identity-defining field wins
(title > author > venue); all wrong fields are listed in the entry's explanation.

### Deliberate exclusions

- **34 real articles** whose *only* substantive error is a wrong (non-future)
  year and/or a wrong volume/issue/page. The work is real and unambiguously
  findable, and the HALLMARK taxonomy has no type for a bare year/volume/page
  slip ‚Äî every such year here is in the past, so `future_date` does not apply.
- **Formatting-only** issues (capitalization, quotes, first-name-vs-initials,
  missing page numbers) are not treated as hallucinations.
- **4 cross-version duplicates** (same work cited by both GPT-3.5 and GPT-4 for a
  topic) and **4 unparseable** rows.

### Result: 341 entries, 172 VALID / 169 HALLUCINATED

All 341 pass `hallmark.contribution.validate_entry.validate_batch`. Built by
`scripts/ingest_walters_wilder.py`; registered in `metadata.json` under
`v1_1_extensions.data_sources`.

## 4. bibtexupdater evaluation

`bibtex-check` (bibtex-updater 1.4.0), `--academic-only`, via the HALLMARK
wrapper (`hallmark/baselines/bibtexupdater.py`) and `hallmark.evaluation.evaluate`.
Full coverage 341/341 (the run took ~4 h of wall-clock under sustained Semantic
Scholar / OpenAlex rate-limiting ‚Äî the documented shared-IP throttling). Reference
output: `data/v1.0/baseline_results/bibtexupdater_raw_supplement_chatgpt_citations.jsonl`;
metrics: `results/walters_wilder/bibtexupdater_metrics.json`.

Before/after comparison is in that section.

|---|---|---|
| Detection rate | **0.929** | [0.905, 0.953] |
| False-positive rate | **0.076** | [0.041, 0.116] |
| F1 (hallucination) | **0.926** | ‚Äî |
| Tier-weighted F1 | **0.959** | ‚Äî |
| ECE | 0.169 | ‚Äî |

Confusion: TP=157, FN=12, FP=13, TN=159.

### Per hallucination type

| Type | count | detection_rate |
|---|---|---|
| `plausible_fabrication` | 139 | **1.000** |
| `wrong_venue` | 5 | 1.000 |
| `near_miss_title` | 13 | 0.769 |
| `swapped_authors` | 12 | **0.250** |

The tool is near-perfect on outright fabrications (its home turf) and weakest on
subtle corruptions of *real* papers: when the title still resolves, it "verifies"
the paper and misses a swapped author or a slightly-off title. This is exactly
the Tier-2/Tier-3 gap HALLMARK is designed to expose ‚Äî `plausible_fabrication`
(T3, DR 1.00) is easy here because a fabricated title simply isn't found, whereas
`swapped_authors` (T2, DR **0.25**) is hard because the work is real and the tool
clears it on a title match without re-checking authorship.

## 5. Error audit and correction

The first run reported DR 0.970 / FPR 0.145. A per-case audit of every false
positive and every miss found that a **single author-parsing bug in the converter
was distorting both**, so the numbers above are the corrected, post-fix figures.

### The bug

`scripts/ingest_walters_wilder.py` split APA reference strings into fields. The
year-split stripped the trailing period off the last author's initial, and the
author regex required each initial to end in a period ‚Äî so the **final author of
every "A, B, & C" citation was silently dropped** (e.g. "Li, H., Yi, J., &
Zhang, J." parsed to only "H. Li and J. Yi"). Separately, APA ellipsis truncation
("first authors ‚Ä¶ last author", used for 8+ authors) was deleted outright, leaving
a short list with no indication it was abbreviated.

Both produced author lists that bibtexupdater read as **silently truncated**, so
it returned `author_truncated`.

### Why it distorted the metrics *both ways*

`author_truncated` maps to HALLUCINATED, so the bug had two opposite effects:

- On **VALID** entries ‚Üí a false positive. 12 of the original 25 FPs were
  `author_truncated`, 8 of them from the dropped-final-author bug.
- On **HALLUCINATED** entries (`swapped_authors`) ‚Üí a *spurious* true positive:
  the tool "detected" the entry for the wrong reason (parser-induced truncation),
  not because it caught the author swap. This inflated the apparent
  `swapped_authors` detection rate to 0.75.

### The fix

1. Allow the last initial to be period-less so the final author is kept.
2. Re-express APA ellipsis / *et al.* truncation as BibTeX `and others` (i.e.
   an explicit, recognised "et al.") instead of deleting it.

Verified on the affected cases: the four sampled `author_truncated` FPs flip to
`verified` (recovered author) or `unconfirmed` (explicit `and others`) ‚Äî both
non-hallucination verdicts.

### Before ‚Üí after (full 341/341 re-run)

| Metric | Before | After |
|---|---|---|
| Detection rate | 0.970 | **0.929** |
| False-positive rate | 0.145 | **0.076** |
| F1 | 0.916 | **0.926** |
| `swapped_authors` DR | 0.75 | **0.25** |
| False positives | 25 | **13** |
| Misses | 5 | **12** |

FPR halved (the artifact FPs are gone) and F1 rose. Detection rate *fell* ‚Äî this
is a **correction, not a regression**: removing the parser-induced truncation
stripped out 7 spurious detections and revealed the tool's true, much weaker
performance on author swaps (`swapped_authors` DR 0.75 ‚Üí 0.25).

### Residual errors are genuine tool behaviour (no further fix)

- **False positives (13):** `title_mismatch` 6 (strict on real-paper
  subtitle/punctuation), `year_mismatch` 4 (online-first vs print year on real
  papers), plus one each `author_mismatch` / `partial_match` / `venue_mismatch`.
  These are the tool's metadata-sensitivity on real works, not parser artifacts.
- **Misses (12):** all `swapped_authors` (9) or `near_miss_title` (3) where the
  tool resolved the real paper by title and abstained (`verified`/`unconfirmed`)
  on the author/title corruption. No wrapper change catches these without hurting
  detection on the main splits (author truncation is a legitimate signal for the
  `partial_author_list` type there), so they are reported, not patched.

## 6. Takeaways

- On authentic ChatGPT **journal-article** citations, bibtexupdater lands at
  **DR 0.93 / FPR 0.076 / F1 0.93** after correcting the converter bug.
- It is essentially a fabrication detector: perfect on invented works
  (`plausible_fabrication` 1.00) but very weak on real works with a swapped author
  (`swapped_authors` **0.25**) ‚Äî it clears them on a title match.
- The audit itself is the headline methodological lesson: a converter bug can
  inflate detection *and* false-positive rates simultaneously, so per-case error
  auditing is necessary before trusting an aggregate score.
- ECE 0.17 indicates the confidence scores remain poorly calibrated.

## 7. Limitations

- **Type-restricted:** articles only. Books/chapters/websites ‚Äî where the tool is
  structurally blind ‚Äî are excluded, so this is not a whole-corpus verdict.
- **APA parsing** is best-effort; long multi-author lists are re-expressed with
  `and others` rather than fully reconstructed.
- **Small per-type cells** (`wrong_venue` n=5, `swapped_authors` n=12) give wide
  CIs; read those detection rates as indicative.

## 8. Reproduction

```bash
# 1. Build the supplement (downloads Appendix 3 if --xlsx omitted)
python scripts/ingest_walters_wilder.py \
    --xlsx 41598_2023_41032_MOESM3_ESM.xlsx \
    --out data/v1.0/supplement_chatgpt_citations.jsonl

# 2. Run bibtexupdater (isolated install; slow under shared-IP rate limits)
uv tool install bibtex-updater
bibtex-check supplement.bib --jsonl results.jsonl --academic-only \
    --rate-limit 60 --workers 4 --cache-file cache.json

# 3. Analyze
python scripts/analyze_walters_wilder.py \
    --supplement data/v1.0/supplement_chatgpt_citations.jsonl \
    --results results.jsonl --xlsx 41598_2023_41032_MOESM3_ESM.xlsx \
    --out-json results/walters_wilder/bibtexupdater_metrics.json \
    --out-md results/walters_wilder/bibtexupdater_report.md
```

(This narrative is hand-maintained; step 3 regenerates only the machine tables
under `results/walters_wilder/`.)
