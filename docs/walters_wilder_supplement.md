# Walters & Wilder ChatGPT-citation supplement — dataset & bibtexupdater analysis

This document describes the `supplement_chatgpt_citations` eval-only split and a
full `bibtexupdater` evaluation on it. Machine-generated metric tables are
regenerated by `scripts/analyze_walters_wilder.py` into `results/walters_wilder/`.

## 1. Source dataset

William H. Walters & Esther Isabelle Wilder, **"Fabrication and errors in the
bibliographic citations generated by ChatGPT"**, *Scientific Reports* 13:14045
(2023), [doi:10.1038/s41598-023-41032-5](https://www.nature.com/articles/s41598-023-41032-5)
(CC BY 4.0). The coded ground truth is Supplementary Appendix 3 (sheet "Works
cited in those papers", 636 rows).

The authors prompted ChatGPT-3.5 and GPT-4 to write 84 short literature reviews
across 42 multidisciplinary topics, then hand-verified every one of the 636
resulting citations, coding each for (a) whether the *work itself* is fabricated
and (b) substantive metadata errors on real works (wrong author, title, year,
journal, volume/pages). Reported fabrication rates: **GPT-3.5 55%**, **GPT-4 18%**.

## 2. Why this is a meaningful HALLMARK extension

HALLMARK's v1.0 splits are almost entirely ML/CS-conference papers produced by
perturbation or scraping. This dataset adds two axes the benchmark is thin on:

1. **Authentic LLM hallucinations** — real ChatGPT output with expert labels,
   not synthetic perturbations. (v1.0 test has only ~90 `llm_generated` +
   ~34 `real_world` entries.)
2. **Multidisciplinary subject matter** — humanities, social sciences, and
   natural sciences, versus HALLMARK's CS/ML focus. It is the natural companion
   to the v1.1 cross-domain split.

## 3. Conversion methodology

Only the **supported publication type — journal articles** — is kept. Books,
chapters and websites (≈32% of the source) are out of scope because DOI /
CrossRef / DBLP / OpenAlex-based verifiers cannot resolve them (in a probe run,
100% of books returned `skipped`).

APA reference strings are parsed into BibTeX fields with a parser hardened for
this corpus's humanities-heavy style: titles containing colons, embedded quotes
or a terminal `?`; APA ellipsis / *et al.* author truncation; and a Chicago-style
bare-year fallback. A quality guard drops any residual broken parse.

### Label mapping (faithful to the HALLMARK taxonomy)

A field mismatch on a real work **is** a hallucination and is mapped to the
matching HALLMARK type rather than left VALID:

| Source coding | HALLMARK type | Tier | n |
|---|---|---|---|
| work itself fabricated | `plausible_fabrication` | 3 | 139 |
| real work, wrong title | `near_miss_title` | 3 | 13 |
| real work, wrong authorship | `author_mismatch` (`swapped_authors`) | 2 | 12 |
| real work, wrong/invented journal | `wrong_venue` | 2 | 5 |
| real work, no substantive error | VALID | — | 172 |

When several fields are wrong, the most identity-defining field wins
(title > author > venue); all wrong fields are listed in the entry's explanation.

### Deliberate exclusions

- **34 real articles** whose *only* substantive error is a wrong (non-future)
  year and/or a wrong volume/issue/page. The work is real and unambiguously
  findable, and the HALLMARK taxonomy has no type for a bare year/volume/page
  slip — every such year here is in the past, so `future_date` does not apply.
- **Formatting-only** issues (capitalization, quotes, first-name-vs-initials,
  missing page numbers) are not treated as hallucinations.
- **4 cross-version duplicates** (same work cited by both GPT-3.5 and GPT-4 for a
  topic) and **4 unparseable** rows.

### Result: 341 entries, 172 VALID / 169 HALLUCINATED

All 341 pass `hallmark.contribution.validate_entry.validate_batch`. Built by
`scripts/ingest_walters_wilder.py`; registered in `metadata.json` under
`v1_1_extensions.data_sources`.

## 4. bibtexupdater evaluation

`bibtex-check` (bibtex-updater 1.4.0), `--academic-only`, via the HALLMARK
wrapper (`hallmark/baselines/bibtexupdater.py`) and `hallmark.evaluation.evaluate`.
Full coverage 341/341 (the run took ~4 h of wall-clock under sustained Semantic
Scholar / OpenAlex rate-limiting — the documented shared-IP throttling). Reference
output: `data/v1.0/baseline_results/bibtexupdater_raw_supplement_chatgpt_citations.jsonl`;
metrics: `results/walters_wilder/bibtexupdater_metrics.json`.

### Headline

| Metric | Value | 95% CI |
|---|---|---|
| Detection rate | **0.970** | [0.947, 0.994] |
| False-positive rate | **0.145** | [0.093, 0.198] |
| F1 (hallucination) | **0.916** | — |
| Tier-weighted F1 | **0.963** | — |
| ECE | 0.206 | — |

Confusion: TP=164, FN=5, FP=25, TN=147.

### Per hallucination type

| Type | count | detection_rate |
|---|---|---|
| `plausible_fabrication` | 139 | **1.000** |
| `wrong_venue` | 5 | 1.000 |
| `near_miss_title` | 13 | 0.846 |
| `swapped_authors` | 12 | **0.750** |

The tool is near-perfect on outright fabrications (its home turf) and weakest on
subtle corruptions of *real* papers: when the title still resolves, it "verifies"
the paper and misses a swapped author or a slightly-off title. This is exactly
the Tier-2/Tier-3 gap HALLMARK is designed to expose — `plausible_fabrication`
(T3, DR 1.00) is easy here because a fabricated title simply isn't found, whereas
`swapped_authors` (T2, DR 0.75) is hard because the work is real.

### By GPT version

| Version | n | detection_rate | FPR |
|---|---|---|---|
| GPT-3.5 | 119 | 0.990 | 0.053 |
| GPT-4 | 222 | 0.928 | 0.157 |

GPT-3.5 is easier on both axes: it fabricates far more (so more easy positives)
and contributes few real citations. GPT-4 produces mostly real citations, so it
offers more opportunities for false positives and its hallucinations are subtler.

### By subject field

| Field | n | detection_rate | FPR |
|---|---|---|---|
| Humanities | 24 | 1.000 | 0.000 |
| Social sciences | 214 | 0.972 | 0.120 |
| Natural sciences | 103 | 0.935 | 0.211 |

### Tool status distribution

`partial_match` 107, `verified` 104, `unconfirmed` 48, `not_found` 34,
`author_truncated` 18, `title_mismatch` 13, `author_mismatch` 8,
`venue_mismatch` 4, `year_mismatch` 3, `hallucinated` 1, missing 1.

## 5. Error analysis

**Misses (5).** All five are subtle corruptions of real papers that the tool
resolved and cleared (`swapped_authors` / `near_miss_title` returned as
`verified`/`unconfirmed`). Author swaps on a real, findable paper are the tool's
blind spot; every outright fabrication was caught.

**False positives (25) — the FPR is inflated by one over-aggressive heuristic.**
Breakdown: `author_truncated` 12, `title_mismatch` 6, `author_mismatch` 2,
`partial_match` 2, `venue_mismatch` 1, `year_mismatch` 1, `not_found` 1.

Nearly half (12/25) are `author_truncated` firing on real articles whose author
lists are **legitimately APA-truncated** (8+ authors → "first authors … last
author"). The cited work is real and correctly identified; flagging the APA
truncation convention as a defect is over-aggressive. Excluding that artifact,
the effective FPR is ≈ **13/172 = 0.076**. The remaining false positives are
`title_mismatch` on real papers whose parsed subtitle/punctuation differs
slightly from the database record.

## 6. Takeaways

- On authentic ChatGPT **journal-article** citations, bibtexupdater is strong:
  **97% detection at 15% FPR** (≈8% discounting the APA-truncation artifact).
- It is essentially a fabrication detector: near-perfect on invented works,
  markedly weaker on real works with a swapped author or near-miss title.
- This distribution exposes two things HALLMARK's CS-conference splits hide: the
  Tier-2 real-paper-corruption gap, and an over-aggressive author-truncation
  heuristic that should be reported separately from genuine detections.
- ECE 0.21 indicates the confidence scores are poorly calibrated (overconfident).

## 7. Limitations

- **Type-restricted:** articles only. Books/chapters/websites — where the tool is
  structurally blind — are excluded, so this is not a whole-corpus verdict.
- **APA parsing** is best-effort; a handful of long multi-author lists are
  shortened, which is the source of the `author_truncated` false positives.
- **Small per-type cells** (`wrong_venue` n=5, `swapped_authors` n=12) give wide
  CIs; read those detection rates as indicative.

## 8. Reproduction

```bash
# 1. Build the supplement (downloads Appendix 3 if --xlsx omitted)
python scripts/ingest_walters_wilder.py \
    --xlsx 41598_2023_41032_MOESM3_ESM.xlsx \
    --out data/v1.0/supplement_chatgpt_citations.jsonl

# 2. Run bibtexupdater (isolated install; slow under shared-IP rate limits)
uv tool install bibtex-updater
bibtex-check supplement.bib --jsonl results.jsonl --academic-only \
    --rate-limit 60 --workers 4 --cache-file cache.json

# 3. Analyze
python scripts/analyze_walters_wilder.py \
    --supplement data/v1.0/supplement_chatgpt_citations.jsonl \
    --results results.jsonl --xlsx 41598_2023_41032_MOESM3_ESM.xlsx \
    --out-json results/walters_wilder/bibtexupdater_metrics.json \
    --out-md results/walters_wilder/bibtexupdater_report.md
```

(This narrative is hand-maintained; step 3 regenerates only the machine tables
under `results/walters_wilder/`.)
