\begin{abstract}
Citation hallucination is not a model problem; it is a workflow problem: authors, reviewers, and venues rarely verify references because no standardized, reliable, automated tool exists, and the NeurIPS 2025 incident in which 53 accepted papers carried fabricated citations is the symptom rather than the cause.
We introduce \textsc{Hallmark} (\textbf{Hall}ucination bench\textbf{mark}), a collection of 2{,}525 BibTeX entries spanning 14 kinds of fabrication (from nonexistent DOIs and invented venues to near-miss title swaps) grouped into three difficulty tiers by how much external lookup they require, with a held-out split whose labels are not released to prevent contamination.
Each entry is a single BibTeX record fed to a verifier, which must return a binary hallucinated-or-valid label together with six structured per-entry checks---DOI resolution, author-list match, venue validity, title match, required-field completeness, and cross-database agreement---so a miss can be traced to the specific check that failed rather than a single opaque score.
Across seven tools covering the full benchmark (a DOI-lookup baseline, five open-weight LLMs, and GPT-5.1), headline accuracy hides a sharp recall--precision split: the DOI baseline is precise but catches only 26\% of hallucinations, while LLMs reach 48--88\% recall with poor calibration (ECE 0.11--0.33), and at the 2\% base rate implied by the NeurIPS 2025 incident the best-calibrated model's positive predictive value is just 8.7\%.
On a separate set of 480 papers from 2024--2025 that postdate every LLM's training cutoff, false positives rise 3--5$\times$ (GPT-5.1: 17\%$\to$73\%) and F1 collapses to near-chance, showing that LLM verifiers fail on recent work for a simple reason---they have not seen it---and \textsc{Hallmark} ships with versioned splits, a baseline registry, fixed scoring code, and a pinned evaluation protocol as an open-source, pip-installable package.
Code and data: \url{https://github.com/anonymous/hallmark}.
\end{abstract}
