\begin{abstract}
Citation hallucination is not a model problem; it is a workflow problem: authors, reviewers, and venues rarely verify references because no standardized, reliable, automated tool exists, and the NeurIPS 2025 incident in which 53 accepted papers carried fabricated citations is the symptom rather than the cause.
We introduce \textsc{Hallmark} (\textbf{Hall}ucination bench\textbf{mark}), a collection of 2{,}525 BibTeX entries spanning 14 kinds of fabrication (from nonexistent DOIs and invented venues to near-miss title swaps) grouped into three difficulty tiers by how much external lookup they require, with a held-out split whose labels are not released to prevent contamination.
Each entry is a single BibTeX record fed to a verifier, which must return a binary hallucinated-or-valid label together with six structured per-entry checks---DOI resolution, author-list match, venue validity, title match, required-field completeness, and cross-database agreement---so a miss can be traced to the specific check that failed.
Evaluating a DOI-lookup baseline, six LLMs (GPT-5.1 and five open-weight models), two agentic GPT-5.1 variants with up to five tool calls per entry (against CrossRef/OpenAlex/arXiv or against bibtex-updater), and bibtex-updater itself, we find that tool access trades one error mode for another: DOI-only catches 26\% of hallucinations; zero-shot LLMs reach 48--88\% recall with poor calibration (ECE 0.11--0.33); a matched lookup budget lifts GPT-5.1's detection from 80\% to 96\%---close to bibtex-updater's 95\%---but inflates false positives $2.7{\times}$ (17\%$\to$46\%), so F1 never beats the zero-shot baseline.
At the 2\% base rate implied by the NeurIPS 2025 incident the best positive predictive value across all settings is 8.7\%, and the rate-limited academic tool HaRC remains infeasible even with a paid Semantic Scholar API key because its per-entry latency is not bounded by quota.
On a separate set of 480 papers from 2024--2025 that postdate every LLM's training cutoff, false positives rise $3$--$5{\times}$ (GPT-5.1: 17\%$\to$73\%) and F1 collapses to near-chance, showing that LLM verifiers fail on recent work for a simple reason---they have not seen it.
\textsc{Hallmark} ships with versioned splits, a baseline registry, fixed scoring code, and a pinned evaluation protocol as an open-source, pip-installable package.
Code and data: \url{https://github.com/anonymous/hallmark}.
\end{abstract}
