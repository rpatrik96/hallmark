\section{Evaluation protocol}
\label{sec:evaluation}

\subsection{Metrics}
\label{sec:metrics}

\textsc{Hallmark} reports five primary metrics and several diagnostic metrics.

\paragraph{Primary metrics.}
\textbf{Detection Rate (DR)} is recall on the hallucinated class.
The \textbf{False Negative Rate (FNR)} $= 1 - \text{DR}$ captures missed hallucinations---a more serious failure mode than false positives, which can be resolved through manual review.
\textbf{False Positive Rate (FPR)} measures valid entries incorrectly flagged; important for deployment where false alarms erode trust.
\textbf{F1-Hallucination} is the harmonic mean of precision and recall on the hallucinated class.

\textbf{Tier-weighted F1 (TW-F1)} weights each hallucinated entry's contribution to precision and recall by its tier (Tier~1: weight 1, Tier~2: 2, Tier~3: 3), rewarding detection of harder hallucinations.
Linear weights reflect the empirical difficulty hierarchy; rankings remain stable across uniform, linear, and quadratic weighting schemes (\cref{app:statistics}).
\textbf{Matthews Correlation Coefficient (MCC)}~\citep{matthews1975} provides a prevalence-invariant summary using all four confusion matrix cells---important because class ratios differ across splits (dev: 56.6\%, test: 65.5\% hallucinated).
\textbf{Expected Calibration Error (ECE)}~\citep{naeini2015} measures how well confidence scores reflect actual accuracy: a tool with ECE\,=\,0.30 reporting 80\% confidence is actually correct only about 50\% of the time.
ECE $< 0.05$ indicates excellent calibration; ECE $> 0.2$ signals unreliable confidence scores.
Formal definitions of TW-F1 and ECE are provided in \cref{app:metric_details}.

\paragraph{Diagnostic metrics.}
\textbf{detect@$k$} measures the fraction of hallucinations detected by at least one of $k$ verification strategies ordered by ascending cost.
\textbf{Per-tier and per-type breakdowns} reveal category-specific strengths.
\textbf{Sub-test accuracy} measures per-sub-test correctness, showing whether tools detect hallucinations for the right reasons.

\subsection{Ranking with incomplete data}
\label{sec:ranking}

Not all tools can process all entries---HaRC completed 1.8\% and verify-citations 6.3\% due to API rate limits.
Standard averaging treats missing predictions as correct, biasing rankings toward low-coverage tools.
We adopt the Plackett-Luce model~\citep{luce1959,plackett1975}---conceptually equivalent to Elo ratings---which estimates tool strength parameters from pairwise outcomes, enabling comparison even when tools were evaluated on different subsets.
Plackett-Luce satisfies independence of irrelevant alternatives by construction, making rankings invariant to tool addition or removal~\citep{hardt2025benchmarks}.
With current coverage (HaRC: 1.8\%, verify-citations: 6.3\%), pairwise overlap is insufficient for stable estimates; we use mean-score ranking as the primary comparison and provide Plackett-Luce infrastructure for future denser evaluations.
Details are in \cref{app:ranking}.
